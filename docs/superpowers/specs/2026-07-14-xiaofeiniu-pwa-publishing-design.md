# 小飞牛计时 PWA 与公网发布设计

日期：2026-07-14

## 目标

将现有“小飞牛计时”手机网页发布为可公开分享、可添加到手机主屏幕、首次联网后可离线使用的 PWA。源码保存到 GitHub，Vercel 作为唯一正式分享入口，并生成指向正式网址的二维码。

## 发布结构

- GitHub 公开仓库：`y111zmkd/xiaofeiniu-timer`。
- Vercel 项目：`xiaofeiniu-timer`。
- GitHub 负责保存源码和版本记录。
- Vercel 负责 HTTPS 公网托管，并作为二维码及日常分享的唯一网址。
- 后续更新继续部署到同一 Vercel 项目，保持正式网址和二维码稳定。

## PWA 组成

现有计算器继续保持纯静态结构，不增加服务器、账号或数据库。新增以下文件和页面连接：

- `manifest.webmanifest`：定义应用名称、启动方式、黑色主题和图标。
- `sw.js`：缓存应用外壳，支持断网重新打开和继续计算。
- `assets/icon-180.png`：iPhone 添加到主屏幕图标。
- `assets/icon-192.png`、`assets/icon-512.png`：安卓及通用 PWA 图标。
- `index.html` 接入清单、Apple 图标、主题颜色和 Service Worker 注册。
- 分享二维码 PNG：编码正式 Vercel 网址，不编码临时预览地址。

## Logo 处理

使用已经选定并保存在 `assets/xiaofeiniu-logo-source.jpg` 的银色牛头时钟图。图标采用同一套视觉，不为苹果和安卓分别设计两版。导出时保持纯黑背景，并将主体缩放到安全区域，避免 iOS、安卓圆角或其他图标遮罩切掉外圈和牛角。

## 离线与更新策略

- 首次使用必须联网完整打开一次，Service Worker 随后缓存首页、清单和图标。
- 断网时导航请求回退到缓存的 `index.html`，计算功能完全在手机本地运行。
- 静态资源采用缓存优先；每次发布影响页面、脚本或图标的版本时递增缓存名称。
- Service Worker 激活后清理旧缓存并接管页面，避免长期保留过期版本。
- 不保存或上传用户输入；应用没有个人数据和云端同步。

## 用户使用流程

1. 用户通过二维码或链接打开 Vercel 正式网址。
2. iPhone 使用 Safari 的“分享 → 添加到主屏幕”；安卓使用 Chrome 的“安装应用/添加到主屏幕”。
3. 主屏幕显示“小飞牛计时”和牛头时钟图标。
4. 首次加载完成后，即使断网也能从桌面图标重新打开并完成两类时间计算。

## 错误与回退

- 如果 GitHub 仓库创建或推送失败，不改变本地源码，先修复认证再重试。
- 如果 Vercel 生产部署失败，不把预览地址写入二维码；先查看构建日志并重新部署。
- 如果浏览器不支持 Service Worker，在线计算仍可正常使用，只是不承诺离线重开。
- 如果移动端仍命中旧缓存，通过递增缓存版本并重新联网打开一次获取新版。

## 验证标准

- 现有航段计算和航时汇总自动测试全部通过。
- `manifest.webmanifest`、Service Worker 和三种图标均可通过 HTTPS 正常访问。
- 在约 390px 手机视口下无横向溢出，滚轮和计算交互正常。
- 默认航段为 `10:01` 至 `11:17`，结果为 `1 小时 16 分钟`。
- 汇总页默认 `1:00 + 1:00`，结果为 `2 小时 0 分钟`，两组滚轮均有选中高亮。
- 在线首次打开后，模拟断网刷新仍能加载页面并计算。
- Vercel 生产网址返回成功状态；二维码扫描后进入相同正式网址。
- GitHub 仓库不包含账号密码、令牌或其他秘密信息。

## 范围边界

本阶段不增加登录、云同步、后台接口、App Store/应用商店发布或电脑端专用界面，也不修改已经确认的计算规则和黑银 UI。
