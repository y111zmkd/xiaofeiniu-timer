# 小飞牛计时离线优先与桌面图标修复设计

## 目标

用户使用可联网的手机流量首次打开 GitHub Pages 版本后，页面明确提示离线准备完成。随后从 iPhone 或安卓主屏幕启动时，应优先使用本地缓存；即使 Wi‑Fi 无法稳定连接 `github.io` 或手机处于飞行模式，两个计算模块仍可打开和使用。主屏幕必须显示现有银色牛头时针 Logo，而不是文字占位图标。

## 设计

- Service Worker 缓存升级为 `xiaofeiniu-shell-v4`。
- 导航请求改为缓存优先：先返回已缓存首页，再在后台联网更新缓存。首次无缓存时才访问网络。
- 首次安装以首页为必须资源；Manifest 和三张手机图标逐项缓存并允许重试，避免单个资源短暂失败导致整个 Service Worker 安装失败。
- 页面等待 Service Worker 激活并确认首页与图标已经缓存后，显示一次“已可离线使用”的状态提示。未准备完成时不误报成功。
- 为苹果触摸图标和 Manifest 图标使用新的版本化文件名，强制手机放弃之前的文字占位图标缓存。
- Manifest 继续使用 GitHub Pages 子目录下的相对 `id`、`start_url` 和 `scope`，不引入根路径依赖。
- 保留现有 GitHub Pages 正式网址、二维码、计算规则和界面结构。

## 更新行为

离线缓存存在时，应用立即显示本地版本。联网情况下，Service Worker 在后台下载新版；用户完全关闭并再次打开桌面应用后使用新版。旧桌面图标需要删除，并从完成离线准备的新版页面重新添加一次。

## 错误处理

- 首次网络完全不可达时无法安装；用户需切换到能访问站点的手机流量。
- 首页未成功缓存时不显示离线成功提示。
- 次要资源缓存失败时进行有限重试；图标未准备完成时不提示用户重新添加主屏幕。
- 已有缓存时，网络超时或连接失败不阻塞启动。

## 验收

- 首次用手机流量打开后出现“已可离线使用”。
- 删除旧桌面入口并重新添加后，主屏幕显示银色牛头时针 Logo。
- iPhone Safari 和安卓 Chrome 从主屏幕联网启动一次后，飞行模式下均能重新启动。
- 已缓存状态下连接无法访问 GitHub Pages 的 Wi‑Fi，应用仍能立即启动。
- 联网发布新版本后，打开一次、完全关闭、再次打开即可更新。
- 两个计算模块、滚轮、清空、加减法及手机宽度布局不回归。
